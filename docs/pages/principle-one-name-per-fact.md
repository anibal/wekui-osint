title:: One name per fact
type:: principle

- Every fact lives in exactly one place. A copy kept elsewhere will eventually disagree with the original, and then there are two answers where there was one question.
- Where it already operates, verbatim:
  - [[decision-2026-07-23-first-query]] — the discovering Query was always the first Appearance's Query: "Two names for one fact is one name too many."
  - [[decision-2026-07-23-follower-count]] — the copied count disagreed with the truth on 34.8% of Posts: "a number that moves every day is a fact about a moment, not about an account."
  - [[decision-2026-07-23-engagement-in-payload]] — the lifted-out copies "saved no work whatsoever."
  - [[decision-2026-07-23-no-folded-text]] — the fold is "produced by the matcher, not stored on the row"; neither dedup folding nor match folding is "a standing property of the Post."
  - [[open-query-state]] — a Query's state is read from timestamps, "which is the meaning"; a stored copy is a separate, later decision.
