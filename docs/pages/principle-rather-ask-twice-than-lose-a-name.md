title:: Rather ask twice than lose a name
type:: principle

- When completeness and economy conflict, completeness wins: we split the question rather than silently drop part of it.
- Where it already operates, verbatim:
  - [[query-text]] — when a Place has more names than one request holds, "its names are split across several Queries rather than quietly dropped: we would rather ask twice than lose a name."
  - [[search]] — slices are always whole, reaching "a little past the window end" rather than leaving the window's edge uncovered.
